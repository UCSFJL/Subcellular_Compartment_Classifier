QuPath Subcellular Compartment Segmentation Workflow
This repository contains a suite of Groovy scripts designed to automate the segmentation of distinct subcellular compartments within QuPath. By systematically defining regions of interest (ROIs), this pipeline enables precise spatial quantification of protein targets across the nucleus, perinuclear space, plasma membrane, and cytoplasm.

Workflow Overview
The pipeline operates sequentially, with each step mathematically relying on the spatial boundaries generated by the previous step.

Nucleus Detection (Manual Prerequisite): The workflow begins by applying a custom QuPath pixel classifier (or standard cell detection) on the DAPI channel to accurately map the initial nuclear boundaries.

Whole Cell Boundary: Defines the absolute outer limits of the cell.

Perinuclear Generation: Expands and contracts the nuclear boundary to map the immediate ring region surrounding the nucleus.

Plasma Membrane Generation: Calculates a standardized boundary wall at the edge of the whole cell region.

Cytoplasm Generation: Subtracts the defined membrane and perinuclear geometries from the whole cell to map the remaining cytoplasmic volume.

Script Execution Order
To avoid geometry calculation errors, run the scripts in the following exact order:

1. cell_boundary_generate.groovy
Function: Identifies and draws the outer boundary for the entire cell (whole_cell annotation).

Prerequisite: Assumes the DAPI-based nucleus space has already been generated.

2. refined_perinuclear_generate.groovy
Function: Generates the perinuclear space. It achieves this by taking the initial nuclear boundary and mathematically extending it slightly inward and outward to capture the immediate surrounding area (typically capturing ER/Golgi networks).

3. refined_PM_generate.groovy
Function: Draws the precise plasma membrane region using the whole_cell annotations as a bounding box (a "cookie cutter"). It shrinks the cell geometry inward by a user-defined thickness and subtracts this inner area from the outer boundary, leaving only the membrane edge.

Important Usage Notes:

Selection Required: You must select your whole_cell annotations in the QuPath hierarchy before running this script. If nothing is selected, the script will throw an error.

Configuration: By default, the script sets the membrane thickness to 1.0 µm (def membraneThicknessMicrons = 1.0). You can edit this variable at the top of the script under the --- SETTINGS --- section to adjust the membrane width for your specific cell type. For touching cells, a 1.0 µm shrink creates a shared 2.0 µm wall.

4. cytoplasm_generate.groovy
Function: Isolates the true cytoplasm space. It utilizes Boolean geometry tools to subtract both the perinuclear and Membrane annotations from the total whole_cell interior.

Troubleshooting & Tips
Calibration: Ensure your image is properly calibrated in QuPath (microns/pixel). If the image lacks calibration metadata, refined_PM_generate.groovy will default to using raw pixels instead of microns, which may severely impact the accuracy of your membrane thickness.

Cell Detections: The plasma membrane script relies on underlying QuPath cell detections. Ensure you have run a cell detection command prior to attempting to generate the membrane walls.
